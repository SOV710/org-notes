* Candidate Selection

- *首选 (1st)*: 按 =Space= 或数字键 =1=
- *次选 (2nd)*: 按 =;= 或数字键 =2=
- *三选 (3rd)*: 按 ='= 或数字键 =3=

* Commit Modes

** 自动上屏 (Auto-commit)

候选唯一 (无重码) 时, 打完全码后无需按空格, 直接自动上屏。多数方案触发码长为四码, 少数为三码或五码 (如猫码, 仓颉)。此功能一般 *不推荐* 依赖。

** 顶屏 (Push-commit)

#
适用于四码方案。全码候选非唯一时, 无需按空格, 直接输入下一字编码即可将上一字首选顶上屏, 可节省一次击键。​*推荐使用*, 纯音码用户需适应习惯切换。

** 标顶 (Punctuation-push)

使用标点 (常用句号, 逗号, 或 =Shift= 组合键如 =Shift+1=) 将首选顶上屏, 省去一次空格。仅适合跟打, 看打等场景, 主动输入时效果欠佳, *不特别推荐*​。

* Typing Metrics

** 击键 (Keystrokes per Second)

每秒敲击键盘次数。以 10 字跟打为基准, KPS > 6 视为合格。

** 键准 (Accuracy)

键准 100% 表示零回改与退格。任何多余的回改或退格操作均会降低键准。

** 码长 (Average Keystrokes per Character)

每字平均用键数 (字均用键)。​*空格亦计入码长*​。

** 键法 (Hand Balance Ratio)

在标准指法前提下, 跟打器统计左右手按键量之比。

- 键法 = 100% : 左右手完美均衡
- 键法 < 100% : 右手占比更多
- 键法 > 100% : 左手占比更多

* Full Code vs. Abbreviated Code

** 全码 (Full Code)

全码指某字在方案中的完整编码, 并非所有方案均为四码定长, "四码方案" 意指编码长度 *不超过* 四码。以虎码为例 :

- =mbbb= → 爽 (四码全码, 最常见)
- =aae= → 乱 (三码全码)
- =wu= → 五 (二码全码)

小鹤音形等方案为严格四码定长。

** 简码 (Abbreviated Code)

简码为全码的前若干码, 书写时附加下划线 =_= 以与全码区分。主要针对高频字词, 目的是降低码长, 提升输入效率。

- *一简* : 全码第 1 码
- *二简* : 全码前 2 码
- *三简* : 全码前 3 码

示例 :

- 行 : 全码 =pxp=, 简码 =p_= (一简)
- 以 : 全码 =vjr=, 简码 =vj_= (二简)
- 直 : 全码 =nqfi=, 简码 =nqf_= (三简)
